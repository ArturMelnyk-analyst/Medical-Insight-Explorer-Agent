# User Guide — Medical Insight Explorer Agent

## Purpose

This guide explains how to use the Medical Insight Explorer Agent locally.

The app allows users to ask natural-language questions about cleaned Medicare healthcare claims data and receive:


- text summaries
- computed analytical results
- optional visualizations

## Start the App

From the project root:

```bash
python app.py
```

The app opens in a local browser window.

If it does not open automatically, use the local URL displayed in the terminal, usually:

```text
http://127.0.0.1:7860
```

## Language Selection

The app supports:

- English
- Deutsch

Use the language dropdown to select the preferred interface language.

The backend analytics engine remains the same in both languages.

## Supported English Questions

Examples:

```text
Show me the shape of all tables
Give me an inpatient summary
Give me an outpatient summary
What is the average beneficiary age?
Show top inpatient providers
Show top outpatient providers
Show inpatient claims by state
What is the diabetes cost summary?
Show reimbursement distribution
```

## Supported German Questions

Examples:

```text
Zeige die Form aller Tabellen
Gib mir eine Zusammenfassung der stationären Claims
Gib mir eine Zusammenfassung der ambulanten Claims
Wie hoch ist das durchschnittliche Alter der Beneficiaries?
Zeige die wichtigsten stationären Provider
Zeige die wichtigsten ambulanten Provider
Zeige stationäre Claims nach Bundesstaat
Wie sieht die Kostenzusammenfassung für Diabetes aus?
Zeige die Verteilung der Erstattungsbeträge
```

## How to Read the Output

Each response contains:

1. **Summary**  
   A short human-readable interpretation of the selected analysis.
2. **Computed result**  
   The actual result returned by deterministic pandas analytics.
3. **Method note**  
   A reminder that the result was computed from cleaned healthcare claims data.
4. **Safety note**  
   A reminder that the app performs claims analytics only.


## Visualizations

Some questions generate charts.

Examples:

- top providers by claim count
- claims by state
- reimbursement distribution
- diabetes reimbursement comparison

Some questions are better answered as text only.

Examples:

- table shape summaries
- inpatient summary
- outpatient summary
- beneficiary age summary

For text-only questions, the app displays a placeholder visualization message.


## Recommended Demo Flow

For a recruiter or reviewer, use this sequence:

1. Select English
2. Ask: Show top inpatient providers
3. Review text output and chart
4. Select Deutsch
5. Ask: Zeige die wichtigsten stationären Provider
6. Review German response and localized chart
7. Ask: Show reimbursement distribution
8. Review histogram output

This demonstrates:

- natural-language interaction
- deterministic analytics
- visualization support
- bilingual UX
- healthcare-safe wording

## Data Requirements

The app expects cleaned Parquet files under:

```text
data/processed/
```
These files are generated by the upstream Healthcare-Data-Cleaning repository.

Full processed datasets are not committed to GitHub.


## Troubleshooting

### App does not start


Check that dependencies are installed:

```bash
pip install -r requirements.txt
```

### Parquet file error

Check that cleaned Parquet files exist under:

```text
data/processed/
```

### Gradio dependency error

Make sure the pinned versions in requirements.txt are installed.


### Charts do not show

Restart the app:

```bash
CTRL + C
python app.py
```
