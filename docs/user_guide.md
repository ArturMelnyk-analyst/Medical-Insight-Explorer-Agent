# User Guide — Medical Insight Explorer Agent

## Purpose

This guide explains how to use the Medical Insight Explorer Agent locally.

The app allows users to ask natural-language questions about cleaned Medicare healthcare claims data and receive:


- text summaries
- computed analytical results
- optional visualizations

## Start the App

From the project root:

```bash
python app.py
```

The app opens in a local browser window.

If it does not open automatically, use the local URL displayed in the terminal, usually:

```text
http://127.0.0.1:7860
```

## Language Selection

The app supports:

- English
- Deutsch

Use the language dropdown to select the preferred interface language.

The backend analytics engine remains the same in both languages.

## Stakeholder Personas

The app includes stakeholder personas to make the demo use cases clearer.

Available personas:

- Hospital Operations Analyst
- Healthcare Fraud Investigator
- Healthcare Policy Researcher

Selecting a persona updates the use-case description and clickable recommended questions.

## How to Read the Output

Each response contains:

1. **Summary**  
   What was computed.

2. **Analytical insight**  
   A cautious, non-causal interpretation of the computed result.

3. **Computed result**  
   The deterministic pandas output.

4. **Method note**  
   A reminder that the result was computed from cleaned healthcare claims data.

5. **Safety note**  
   A reminder that the app performs claims analytics only.


## Visualizations

Some questions generate charts.

Examples:

- top providers by claim count
- claims by state
- reimbursement distribution
- diabetes reimbursement comparison

Some questions are better answered as text only.

Examples:

- table shape summaries
- inpatient summary
- outpatient summary
- beneficiary age summary

For text-only questions, the app displays a placeholder visualization message.


## Recommended Demo Flow

1. Select English
2. Select Hospital Operations Analyst
3. Click `Show top inpatient providers`
4. Run analysis
5. Review the analytical insight and chart
6. Switch to Deutsch
7. Select the German operations persona
8. Click `Zeige die wichtigsten stationären Provider`
9. Run analysis
10. Review the German analytical insight and chart

## Data Requirements

The app expects cleaned Parquet files under:

```text
data/processed/
```
These files are generated by the upstream Healthcare-Data-Cleaning repository.

Full processed datasets are not committed to GitHub.


## Troubleshooting

### App does not start


Check that dependencies are installed:

```bash
pip install -r requirements.txt
```

### Parquet file error

Check that cleaned Parquet files exist under:

```text
data/processed/
```

### Gradio dependency error

Make sure the pinned versions in requirements.txt are installed.


### Charts do not show

Restart the app:

```bash
CTRL + C
python app.py
```
