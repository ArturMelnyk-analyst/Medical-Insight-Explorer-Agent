# Limitations

## Purpose

This document explains the current limitations of the Medical Insight Explorer Agent.

The goal is to be transparent about what the project can and cannot do.

## Claims Analytics Only

This project analyzes healthcare claims data.

It does not provide:


- diagnosis
- treatment recommendations
- clinical decisions
- patient-level medical advice
- causal medical conclusions

## Limited Supported Questions

The current version supports controlled analytics routes only.

Supported examples include:

- table shape summaries
- inpatient summaries
- outpatient summaries
- beneficiary age statistics
- top provider rankings
- claim distribution by state
- diabetes reimbursement summaries
- reimbursement distributions

Unsupported questions return fallback responses.

## No Free DataFrame Manipulation by LLM

The LLM layer does not directly manipulate pandas dataframes.

This is intentional.

All numerical results are computed using deterministic analytics functions before explanation.

## Local Data Dependency

The app depends on cleaned Parquet files generated by the upstream `Healthcare-Data-Cleaning` repository.

Full processed datasets are excluded from GitHub.

A lightweight sample-data package may be added later for demo deployment.

## Dataset Scope

The project is based on cleaned Medicare healthcare claims data from the original project pipeline.

The dataset may not represent:

- all healthcare systems
- current real-world claims behavior
- clinical reality
- causal medical relationships

## Diabetes Reimbursement Comparison

The diabetes reimbursement comparison is included as an example of controlled chronic-condition analytics.

In the current dataset, the reimbursement distributions for beneficiaries with and without diabetes appear highly similar. This means the visualization should be interpreted as a distribution comparison rather than as evidence of a strong reimbursement difference.

The project does not claim that diabetes status causes higher or lower reimbursement values.

## Bilingual Limitations

The app supports English and German user-facing text for selected analytical routes.

German prompts are normalized into supported deterministic analytics routes before computation.

This is not full natural-language understanding for arbitrary German questions.

## Visualization Limitations

Charts are generated for selected question types only.

Some analytical questions are intentionally text-only.

Chart routing is currently simple and keyword-based.

## Deployment Limitations

The current implementation primarily targets local execution.

Hugging Face Spaces deployment is planned as a future step after sample data is prepared.
