# Visualization Tools

## Purpose

The visualization layer converts deterministic healthcare analytics outputs into interactive Plotly charts.

Rather than allowing future LLM components to generate arbitrary visual interpretations, this module creates charts directly from validated outputs produced by the `HealthcareAnalyticsEngine`.

This design reduces hallucination risk and keeps visual outputs grounded in real computed healthcare metrics.

---

## Current Visualization Functions

| Function | Purpose |
|-----------|----------|
| `bar_chart_top_values()` | Creates vertical bar charts for ranked or grouped healthcare metrics |
| `horizontal_bar_chart()` | Creates horizontal bar charts for long category labels such as Provider IDs |
| `histogram()` | Creates histograms for numeric distributions |
| `box_plot()` | Creates box plots for comparing distributions across healthcare groups |
| `validate_columns()` | Performs input validation before chart generation |

---

## Architecture

Visualization functions do not directly query datasets.

They operate only on outputs generated by the deterministic analytics layer.

Pipeline:

```text
User Question
      ↓
HealthcareDataLoader
      ↓
HealthcareAnalyticsEngine
      ↓
Visualization Tools
      ↓
Plotly Figure
      ↓
Future Gradio Interface
```

This architecture ensures:

deterministic calculations
reusable chart generation
reduced hallucination risk
separation of responsibilities
future compatibility with LLM orchestration

## Design Notes

Local Jupyter rendering uses:

import plotly.io as pio

pio.renderers.default = "browser"

This setting improves local visualization reliability.
Future Gradio deployment will render Plotly figures directly within the application interface.

## Scope

This module currently provides chart-generation utilities only.

Automatic chart selection, natural-language routing, and LLM-driven visualization requests will be implemented in later project phases.
