\# Data Contract — Medical Insight Explorer Agent



\## Purpose



This document defines the expected cleaned Parquet inputs for the Medical Insight Explorer Agent.



The project consumes validated outputs generated by the upstream `Healthcare-Data-Cleaning` repository. It does not start from raw CSV files.



\## Expected Data Location



Full processed Parquet files should be placed locally under:



```text

data/processed/

```

These files are intentionally excluded from GitHub.



\## Required Parquet Tables



| Logical Table Name  | Expected File                     | Role                                     |

| ------------------- | --------------------------------- | ---------------------------------------- |

| `train\_beneficiary` | `train\_beneficiary\_clean.parquet` | Cleaned training beneficiary table       |

| `test\_beneficiary`  | `test\_beneficiary\_clean.parquet`  | Cleaned test beneficiary table           |

| `train\_inpatient`   | `train\_inpatient\_clean.parquet`   | Cleaned training inpatient claims table  |

| `test\_inpatient`    | `test\_inpatient\_clean.parquet`    | Cleaned test inpatient claims table      |

| `train\_outpatient`  | `train\_outpatient\_clean.parquet`  | Cleaned training outpatient claims table |

| `test\_outpatient`   | `test\_outpatient\_clean.parquet`   | Cleaned test outpatient claims table     |

| `train\_labels`      | `train\_labels\_clean.parquet`      | Training provider-level label table      |

| `test\_labels`       | `test\_labels\_clean.parquet`       | Test provider-level label table          |



\## Key Relationship Fields



| Field      | Purpose                                                         |

| ---------- | --------------------------------------------------------------- |

| `BeneID`   | Connects beneficiary records to inpatient and outpatient claims |

| `ClaimID`  | Identifies individual claim records                             |

| `Provider` | Connects claim records to provider-level label tables           |



\## Data Governance



Full processed datasets are not committed to GitHub.



The expected workflow is:



1. Generate cleaned Parquet files in the upstream Healthcare-Data-Cleaning project.

2\. Copy the generated Parquet files locally into data/processed/.

3\. Use HealthcareDataLoader to load the relational tables into the AI Agent project.
